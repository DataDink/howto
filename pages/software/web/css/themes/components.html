Components

Components, not to be confused with web components, refer to the more specific parts of the application like lists, blockquotes, etc. These can be broken out from the theme.css and layout.css allowing them to be more modular. This might also contain styles for <code> elements and <pre> elements. Depending on the application, I might break these out into separate files or include more block-type elements in the same file. For me, these divisions become very situational. It's not as cut & dry as I like most things to be, but as long as I keep "single-responsibility" and "separation-of-concerns" in mind, I can always break things out later with minimal refactoring efforts.
